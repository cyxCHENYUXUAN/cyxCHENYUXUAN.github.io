图的遍历：BFS与DFS详解，应用场景

发表于 2025-04-06 分类于 数据结构 阅读次数：
本文字数： 5.5k 阅读时长 ≈ 5 分钟

  • 遍历的定义：从已给的连通图中某一个顶点出发，沿着边访问图中所有的顶点，且每个顶点仅被访问一次，就叫做图的遍历，它是图的基本运算

  • 遍历的实质：找每个顶点的邻接节点的过程

  • 图的特点：图中可能存在回路，且图的任一顶点都可能与其它顶点相通，在访问完某个顶点之后，可能会沿着某些边又回到了曾经访问过的顶点

  • 如何避免不重复访问

    解决思路——设置辅助数组$visited[n]$来标记每个顶点是否已经被访问

    • 初始状态$visited[i]$为$0(false)$
    • 顶点$i$被访问，将$visited[i]$改为$1(true)$，防止其被多次访问

  • 图的常见的遍历算法

    • 广度优先搜索（BFS）
    • 深度优先搜索（DFS）

一. 广度优先搜索（BFS）

图的广度优先搜索（BFS）类似于树的层次遍历

1. 基本思想

首先访问起始顶点$v$，然后选取与$v$邻接的全部顶点$w$，进行访问，再依次访问与$w$邻接的全部顶点（已经访问过的除外），以此类推，直到所有顶点都被访问过为止。

  • 广度优先搜索遍历图的时候，需要用到一个队列（二叉树的层次遍历也要用到队列），节点入队时，将该节点设置为已访问，执行过程概括如下：
    • 任取图中的一个顶点访问，将其入队，将这个顶点标记为已访问。
    • （当队列不为空时循环执行此次操作）将队头顶点出队，检查出队顶点的所有邻接顶点，访问未被访问过的邻接顶点，同时将其入队。
    • 当队列为空时，跳出循环，BFS完成。

2. BFS的性能分析

无论是邻接表还是邻接矩阵的存储方式，BFS算法都需要借助一个辅助队列，$n$个顶点均入队一次，在最坏的情况下，空间复杂度为$O(|V|)$。

遍历图的过程实质上是对每个顶点查找其邻接点的过程，耗费时间取决于所采取的存储结构。

采用邻接表存储，每个顶点均需要搜索（入队）一次，时间复杂度为$O(|V|)$，在搜索每个顶点的邻接点时，每个边至少访问一次，时间复杂度为$O(|E|)$，总的时间复杂度为$O(|V|+|E|)$。

采用邻接矩阵存储，查找每个顶点的邻接点所需的时间为$O(|V|)$，查找对应的边需要的时间为$O(|V|^2)$，总时间为$O(|V|^2)$。

3. 广度优先生成树

在广度遍历的过程中，我们可以得到一棵遍历树，称为广度优先生成树，如图所示。需要注意的是，一给定图的邻接矩阵存储表示是唯一的，故其广度优先生成树也是唯一的，但由于邻接表存储表示不唯一，故其广度优先生成树也是不唯一的。

4. BFS算法的应用场景——求（无权图的单源）最短路径问题

由于广度优先搜索总是按照距离由近到远遍历图中的每个顶点，所有可以利用这个性质求解最短路径问题。

二. 深度优先搜索（DFS）

图的深度优先搜索（DFS）类似于树的先序遍历

1. 基本思想

首先访问图中某一起始地点$v$，然后由$v$出发，访问与$v$邻接且未被访问过的任意一个顶点$w_1$，再访问与$w_1$邻接且未被访问过的任意一个顶点$w_2$……重复上述过程。当不能再继续向下访问时，依次退回到最近被访问的顶点，若它还有邻接顶点未被访问过，则从该点开始继续上述搜索，直到图中所有顶点被访问过为止。

  • 深度优先搜索遍历图时，需要用到一个栈（或采用递归），栈顶节点出栈，此时标记该节点为已访问，算法执行过程如下：
    • 首先访问顶点$i$，并将其设置为已访问，即$visited[i]=1$。
    • 然后搜索与顶点$i$有边相连的下一个顶点$j$，若$j$未被访问过，则访问它，并且将$j$设置为已访问，$visited[j]=1$，然后从$j$开始重复此过程，若$j$已访问，则查看$i$的其它邻接节点。
    • 若$i$的所有邻接节点均被访问过，则回退到前一个顶点并且重复上面的第二步，直到所有顶点被访问完。

下图展示了DFS的过程（节点上的蓝色数字代表访问顺序）

2. DFS的性能分析

DFS算法是一个递归算法，需要借助一个递归工作栈；所以其空间复杂度为$O(|V|)$。

遍历图的过程实质上是通过边来查找邻接点的过程，因此两种遍历方式（DFS和BFS）的时间复杂度都相同，不同之处在于对顶点的访问顺序不同。采用邻接矩阵存储时，二者的时间复杂度均为$O(|V|^2)$；采用邻接表存储时，二者的时间复杂度均为$O(|V|+|E|)$。

3. 深度优先生成树和生成森林

对连通图调用DFS才能产生深度优先生成树，否则将产生深度优先生成森林。与BFS类似，基于邻接表存储的深度优先生成树是不唯一的。

4. DFS算法的应用场景——拓扑排序、连通分量检测、迷宫问题

深度优先遍历主要思路是从一条未访问的顶点V开始，沿着一条路一直走到底，然后从这条路尽头的节点回退到上一个节点，再从另一条路开始走到底……，不断循环重复这个过程，直到所有的顶点都遍历完成，它的特点是不撞南墙不尽头，先走完一条路，换再继续一条路走。

DFS沿着路径尽可能深入，回溯后继续探索其它分支。利用这种特点，可以解决上述问题。

三. 图的遍历和图的连通性

对于无向图，调用DFS()和BFS()（这两个函数在下文给出）的次数等于该图的连通分量数。

在有向图中，则不是这样的。

图的遍历算法可以用来判断图的连通性。

对于无向图：

  • 若无向图是连通的，则从任一结点出发，仅需一次遍历就能够访问图中的所有顶点
  • 若无向图是非连通的，则从某一个顶点出发，一次遍历只能访问到该顶点所在连通分量的所有顶点，而对于图中其他连通分量的顶点，则无法通过这次遍历访问。

对于有向图：

  • 若从初始顶点到图中的每个顶点都有路径，则能访问到图中的所有顶点，否则不能访问到所有顶点。
