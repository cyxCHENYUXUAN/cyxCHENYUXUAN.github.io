最长公共子序列

力扣题目链接

给定两个字符串 text1 和 text2，返回这两个字符串的最长 公共子序列 的长度。如果不存在 公共子序列 ，返回 0 。

一个字符串的 子序列 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。

  • 例如，"ace" 是 "abcde" 的子序列，但 "aec" 不是 "abcde" 的子序列。

两个字符串的 公共子序列 是这两个字符串所共同拥有的子序列。

示例 1：

输入：text1 = “abcde”, text2 = “ace”
输出：3
解释：最长公共子序列是 “ace” ，它的长度为 3 。

示例 2：

输入：text1 = “abc”, text2 = “abc”
输出：3
解释：最长公共子序列是 “abc” ，它的长度为 3 。

示例 3：

输入：text1 = “abc”, text2 = “def”
输出：0
解释：两个字符串没有公共子序列，返回 0 。

什么是动态规划

动态规划能够把原问题分解为相对简单的子问题来求解复杂的问题的方法。

核心思想如下：

  • 将复杂问题分解为相互关联的子问题
  • 通过解决子问题间接解决原问题
  • 保存已解决的子问题的结果

什么时候（出现什么特征）能够使用动态规划方法：

  • 最优子结构：总问题的最优解用到子问题的最优解
  • 重叠子问题：求解总问题时，相同的子问题重复出现
  • 状态转移：找出子问题之间的联系，通过状态转换方程来解决问题

解决动态规划问题的思路、方法：

  • 定义状态：用数学语言描述问题，思考要解决的问题需要使用哪些变量。得到状态的数学定义，这些状态一般以数组的形式存储，数组一般叫做$dp$数组。
  • 找出状态转移方程：关键——如何通过已知的状态推导出新的状态，状态转移方程就是如何从已知状态推导出新状态的数学语言描述，一般找到状态转移方程就可以很方便的解决问题。
  • 确定初始状态和边界条件：也就是特殊情况——数组边界或者输入为$0$或负数（具体情况具体分析）。
  • 按照状态转移方程求解：也就是实际求解的过程。按照一定的顺序（比如从小到大）来计算每个状态的值。在这个过程中，使用已经计算出的状态值，通过状态转移方程来得到新的状态值。这个过程需要特别注意计算的顺序，确保在计算某个状态时，它依赖的所有状态都已经计算出来。

解题思路

1. 案例分析

先来看个案例，从中找到状态的定义和状态转移方程（有时遇到动规的题目无从下手，可以先举个例子来分析）
以 X = “ABCBDAB” 和 Y = “BDCAB” 为例

  • 首先定义一个$dp$二维数组（对应着两个字符串），明确这个数组的含义。数组大小为$[X.size() + 1]*[Y.size() + 1]$，此案例中$X.size()$为$7$，$Y.size()$为$5$。

    此题中，$dp$数组的含义：对于$X$和$Y$的子字符串$X[1…i]$和$Y[1…j]$，某处（设为$(i, j)$）二者的最长公共子序列（$LCS$）就是$dp[i][j]$。

    对于上方的表：

    • 红色元素：表示边界条件。索引为$0$的行和列表示空串，dp[0][..] 和 dp[..][0] 都应该初始化为 0，这就是$base case$。

    • 黑色元素：表示此时$X[i] != Y[j]$，即当前字符不匹配。则LCS 的长度由以下两种情况中的较大值决定。

      • 忽略 X 的第 i 个字符：比较 X 的前 i-1 个字符与 Y 的前 j 个字符的 LCS，即 dp[i-1][j]。
      • 忽略 Y 的第 j 个字符：比较 X 的前 i 个字符与 Y 的前 j-1 个字符的 LCS，即 dp[i][j-1]。

      此时状态转换方程：

    • 蓝色元素：表示此时$X[i] == Y[j]$，即当前字符匹配，则该字符必定属于 LCS。这时 LCS 的长度为 $X$的前 $i-1$个字符与 $Y$ 的前$j-1$个字符的LCS长度上当前匹配的字符。

      此时状态转换方程：

  • 接下来，从$Y[1]$开始遍历$X[1…X.size()]$（或者从$X[1]$开始遍历$Y[1…Y.size()]$）

    • 首先访问$Y[1]$，也就是字符B

      当在$X$中遍历到$X[2]$、$X[4]$时，$X[i] == Y[j]$，也就是字符串匹配的情况：

      字符相等 → $dp[1][2] = dp[0][1] + 1$、$dp[1][4] = dp[0][3] + 1$

      当在$X$中遍历到其它字符时，$X[i] != Y[j]$，也就是字符串不匹配的情况：

      字符不等 （以遍历到$X[3]$为例）→ $dp[1][3] = max(dp[1][2], dp[0][3])$

    • 接下来访问$Y[2]$，也就是字符D（注意：此时需要保存访问过的$Y[1]$的状态）

      当在$X$中遍历到$X[5]$时，$X[i] == Y[j]$，也就是字符串匹配的情况：

      字符相等 → $dp[2][5] = dp[1][4] + 1$（注意看这时就用到了访问过的$Y[1]$中的$dp[1][4]$，那么后面也会用到此时访问$Y[2]$时用到的$dp[2][5]$）

      当在$X$中遍历到其它字符时，$X[i] != Y[j]$，也就是字符串不匹配的情况：

      字符不等 （以遍历到$X[3]$为例）→ $dp[2][3] = max(dp[2][2], dp[1][3])$

    • 下面访问$Y[3]、Y[4]…Y[5]$的情况都与上面类似，不再赘述

2. 思路总结

通过上面的案例，我们已经了解了$dp$数组的含义、状态转移方程、边界条件，如下：

  • $dp$数组的含义

    对于$X$和$Y$的子字符串$X[1…i]$和$Y[1…j]$，某处（设为$(i, j)$）二者的最长公共子序列（$LCS$）就是$dp[i][j]$

  • 状态转移方程

    某处字符串匹配时——$dp[i][j]=dp[i−1][j−1]+1$

    某处字符串不匹配——$dp[i][j]=max(dp[i−1][j],dp[i][j−1])$

  • 边界条件

    当 $i=0$或 $j=0$时，$dp[i][j] = 0$，因为空序列没有公共子序列
